# Markdrop 手动测试清单

修改捕获逻辑、Markdown 转换、保存集成、配置页或扩展权限后，请跑这份清单。

## 构建

- 运行 `npm run typecheck`。
- 运行 `npm run build`。
- 在 `chrome://extensions` 或 `edge://extensions` 重新加载 `dist/`。
- 确认配置页底部显示预期 build ID。

## 全新浏览器安装

- 在干净 Chrome 或 Edge 配置中加载 `dist/`。
- 打开 Markdrop 配置页。
- 确认 Notion、Feishu、Obsidian 三个平台卡片可见。
- 确认必填字段带 `*`。
- 确认没有预填真实凭据。
- 切换语言偏好：`Auto`、`中文`、`English`，确认配置页和保存弹窗文案同步变化。

## 普通网页划词保存

- 在普通网页中选中文本。
- 使用浏览器右键菜单 `Save to Markdrop`。
- 确认保存弹窗打开。
- 确认标题来自网页标题。
- 分别保存到已配置的目标平台。

## AI 平台按钮

每个平台确认每个回答下方只出现一个保存按钮，且不会插入到思考过程、复制工具栏或页面悬浮容器中。

- ChatGPT
- Claude
- Gemini
- DeepSeek
- Doubao
- Qianwen

每个平台保存一段包含下列格式的代表性回答：

- 标题
- 段落
- 加粗和斜体
- 行内代码
- 带语言标识的代码块
- 表格
- 有序和无序列表，包括嵌套列表
- 任务列表
- 引用
- 行内和块级公式
- 分割线

## AI 页面划词保存

- 选中 AI 回答中的一小段内容。
- 保存该选区。
- 确认公式和代码块没有被压扁或吞入其它代码块。
- 确认保存标题仍使用对应 AI 回答/页面标题。
- 确认选区内容没有误包含相邻回答。

## Notion

- 验证 Notion Page 保存位置。
- 验证 Notion Data Source 保存位置。
- 保存完整 AI 回答到 Page。
- 保存 AI 回答选区到 Page。
- 确认公式可读。
- 确认支持的 Markdown 表格保存为表格。
- 确认代码块保留换行和语言标签。
- 当目标没有授权给 integration 时，错误提示应清楚说明需要分享/连接目标。

## Feishu

- 完成 OAuth 登录。
- 保存到云文档文件夹。
- 保存到 Wiki 目录。
- 确认 Markdown 表格创建为原生表格。
- 确认公式可读，无法原生渲染时也要降级为可读文本。
- 确认代码块保留换行。
- 当 refresh token 失效时，提示用户重新登录。

## Obsidian

- 验证 Local REST API 保存位置。
- 保存到例如 `AI摘录` 的 Vault 文件夹路径。
- 确认文件写入预期文件夹。
- 配置 Vault 名称后，确认 `obsidian://` 打开链接可用。
- 确认代码围栏正确闭合，后续内容没有被吞进代码块。
- 确认代码块保留语言标签，Obsidian 预览中有高亮。
- 确认任务列表保存为 `- [x]` / `- [ ]`。
- 确认公式在 Obsidian 预览中可读。

## 回归记录

如果某个平台保存结果和原网页差异明显，请记录：

- 来源平台和 URL
